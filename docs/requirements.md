# VR 건설 현장 안전 점검 역량 평가 시뮬레이션 — 요구사항 정의서

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| **프로젝트명** | VR 건설 현장 안전 점검 역량 평가 시뮬레이션 |
| **목적** | 현장 관리자의 위험 요소 탐지 역량 평가 (시험용) |
| **맥락** | 대학 연구 프로젝트 |
| **사용자** | 신입·경력 현장 관리자 (1명씩 개별 진행) |
| **핵심 제약** | 연구 특성상 요구사항이 동적으로 변경됨 → 확장 가능한 설계 필수 |

## 2. 시뮬레이션 환경

### 2.1 현장 유형

- 건물 골조 공사 현장 (철근콘크리트 구조물)

### 2.2 위험 요소

> 구체적 항목은 연구 진행에 따라 점진적으로 확장 예정

- 크랙 (균열)
- 붕괴 조짐
- 철근 부식
- 누수
- 지반 침하
- 가시설 변형
- 안전 장비 미비

### 2.3 난이도

- 연구 목적에 따라 난이도 가변 (예: 명확한 크랙 ~ 미세 크랙)
- 단계별 체계보다는 연구 조건에 맞춰 유연하게 조정

### 2.4 3D 에셋

- 기존 에셋 없음
- 실제 참고 사진을 기반으로 절차적(procedural) 생성

## 3. 사용자 인터랙션

| 항목 | 방식 |
|---|---|
| **이동** | 컨트롤러 조이스틱 |
| **위험 요소 발견** | 대상을 바라보고 컨트롤러 버튼으로 마킹 |
| **시선 추적** | 화면 중심 기반 (화면 중심부를 보고 있다고 가정) |

## 4. 평가 방식

### 4.1 평가 지표

| 지표 | 설명 |
|---|---|
| **발견율** | 제한 시간 내 위험 요소를 얼마나 많이 찾았는지 |
| **반응 시간** | 위험 요소를 얼마나 빨리 찾았는지 |

### 4.2 뇌파(EEG) 연동

- 뇌파 측정 기기: 미정
- 연동 방식: 각각 독립 기록 후 **타임스탬프 기반 후처리 동기화**
  - VR 시뮬레이션: 이벤트 로그에 타임스탬프 기록
  - EEG 기기: 별도 소프트웨어에서 독립 기록
  - 후처리 단계에서 타임스탬프를 기준으로 두 데이터를 동기화

## 5. 데이터 수집 및 저장

### 5.1 저장 방식

- 로컬 파일 저장 (CSV / JSON)
- 후처리 분석

### 5.2 기록 항목

| 분류 | 항목 |
|---|---|
| **피험자 정보** | ID, 경력 등 |
| **세션 설정** | 난이도, 시간 제한, 위험 요소 배치 정보 |
| **위험 요소별** | 발견 여부, 발견 시각, 마킹 시 사용자 위치, 마킹 시 시선 방향 |
| **사용자 행동** | 이동 경로 (주기적 샘플링), 시선 방향 |
| **오탐** | 위험이 아닌 곳을 마킹한 기록 |

### 5.3 뇌파 동기화

- VR 로그와 EEG 로그의 타임스탬프를 맞춰 후처리 분석

## 6. 시나리오 관리

- 연구자가 **코드 수정 없이** 위험 요소 배치를 변경할 수 있는 설정 파일 제공
- 위험 요소 **랜덤 배치** 기능 지원
- 시나리오 설정 파일 예시 항목:
  - 위험 요소 종류
  - 위치
  - 난이도 (크기, 명확도 등)

## 7. 세션 흐름

> 구체적 흐름은 추후 설계 예정

- 시간 제한 있음
- 기본 흐름 (잠정):
  1. 피험자 정보 입력
  2. 시뮬레이션 진행
  3. 결과 기록 및 저장

## 8. 기술 스택

| 항목 | 선택 |
|---|---|
| **VR 하드웨어** | Meta Quest 시리즈 |
| **개발 엔진** | Godot 4 (GDScript + OpenXR) |
| **개발 방식** | Claude Code 기반 전체 개발 |
| **씬 형식** | `.tscn` (텍스트 기반 — Claude Code 친화적) |
| **데이터 저장** | 로컬 파일 (CSV / JSON) |

### 8.1 Godot 4 선정 사유

- `.tscn` 씬 파일이 사람이 읽을 수 있는 텍스트 형식 → Claude Code로 직접 생성·편집 가능
- GDScript가 Python과 유사하여 생산성 높음
- OpenXR 기반 Meta Quest 지원
- 오픈소스 → 연구 프로젝트에 적합
- 확장 가능한 아키텍처

## 9. MVP (최소 기능 제품)

> 건물 골조 현장을 조이스틱으로 돌아다니며, 배치된 크랙 약 3개를 컨트롤러 버튼으로 마킹하여 발견하는 시뮬레이션

### MVP 포함 범위

- [ ] 건물 골조 현장 3D 환경 (기본 구조물)
- [ ] 조이스틱 기반 이동
- [ ] 크랙 위험 요소 3개 배치
- [ ] 컨트롤러 버튼 마킹으로 발견 처리
- [ ] 발견율 및 반응 시간 기록
- [ ] 세션 결과 로컬 파일 저장

## 10. 향후 확장 고려 사항

- 위험 요소 종류 추가 (철근 부식, 누수, 지반 침하 등)
- 현장 유형 추가 (터널, 도로, 교량 등)
- 뇌파 기기 실시간 연동 (LSL 등)
- 시선 추적 고도화 (HMD 내장 eye tracker 활용)
- 시나리오 에디터 UI
- 다양한 난이도 프리셋
